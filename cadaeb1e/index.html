题目传送门

思路

由题意得，每次从 i 可以跳一步到最小的点就是 l[i] .

由于是双向的，因此只要满足 l[j]<=i(j>i)l[j]<=i(j>i)l[j]<=i(j>i) 那么便可以从 jjj 一步跳到 iii ，记满足条件的最大的 jjj 为 rmaxrmaxrmax .

那么定义：

fi,0f_{i,0}fi,0​ 代表从 iii 开始走 202^020 次可以到达的最小的点。

那么 fi,jf_{i,j}fi,j​ 代表从 iii 开始走第 2j2^j2j 次可以到达的最小的点。

fi,0=l[i]f_{i,0}=l[i]fi,0​=l[i] ，也就是从 iii 走 0(20)0(2^0)0(20) 次所能到达的最小的点。

再走一步呢？

那就是 fi,1=mink=linlkf_{i,1}=min^n_{k=l_i}l_kfi,1​=mink=li​n​lk​ .

为什么是从 iii 跳一步所能到的最左端的点到 nnn 这个区间内寻找更小的左端点呢？

为什么不是到 rmaxrmaxrmax 呢？

其实就是 rmaxrmaxrmax ，但是 rmaxrmaxrmax 的左端点已经是满足条件最大的了，再往右，也就是从 rmax+1rmax+1rmax+1 到 nnn 都不会满足条件，也就是说这些点连 iii 都一步跳不到，怎么可能一步跳到比 l[i] 还小的点呢？

为了方便，所以写到 nnn ，这样就不用特意计算 rmaxrmaxrmax 了。

所以递推我们得到:

fi,j=ffi,j−1,j−1f_{i,j}=f_{f_{i,j-1},j-1} fi,j​=ffi,j−1​,j−1​

也就是说，从 iii 开始走第 jjj 次可以到达的最小的点就是从 iii 开始走 2j−12^{j-1}2j−1 次可以到达的最小的点到所有点之间的这些点，从这些点再跳 2j−12^{j-1}2j−1 步所能到达的最小的点。

这就有 倍增 的意思了。

但是，但是：

连续向左跳可不一定比先向右再向左跳更优。

所以 fi,0=min(l[i],fi+1,0)f_{i,0}=min(l[i],f_{i+1,0})fi,0​=min(l[i],fi+1,0​) ，其中fn,0f_{n,0}fn,0​一定为l[n]l[n]l[n]。

这个需要倒着往前推，这样就可以都保证是最优解

对于从 nnn 开始的第一步，它所能到达的最小的点只能是它的 lll 值，因为它没法向右了。

这样的话从 iii 右边的 jjj 号点先跳到 iii 还是先跳到 i+1i+1i+1 ，下一步都是一样的，因为贪心的缘故，假如 i+1i+1i+1 往后跳更优，也就是 fi+1,0f_{i+1,0}fi+1,0​ 更小，那么 iii 的下一步将和 i+1i+1i+1 的下一步一致（因为 fi,0f_{i,0}fi,0​ 的时候有个取 min 操作）。

根据答案形式为求区间期望，考虑前缀和优化。

用 sumi,jsum_{i,j}sumi,j​ 代表从 iii 到 fi,j→(i−1)f_{i,j}\to{(i-1)}fi,j​→(i−1) 这些点的最小步数之和，也就是从 iii 转移 2j2^j2j 步后所到达的最小的点到 i−1i-1i−1 这段区间内到 iii 的最小步数之和（步数=转移次数）

转移：

sumi,0=i−fi,0sum_{i,0}=i-f_{i,0} sumi,0​=i−fi,0​

因为只用跳一次，区间内点的个数即转移代价。

sumi,j=sumi,j−1+sumfi,j−1,j−1+2j−1×(fi,j−1−ffi,j)sum_{i,j}=sum_{i,j-1}+sum_{f_{i,j-1},j-1}+2^{j-1}\times({f_{i,j-1}-f_{f_{i,j}}}) sumi,j​=sumi,j−1​+sumfi,j−1​,j−1​+2j−1×(fi,j−1​−ffi,j​​)

因为分了两次来跳，其实就是点的个数再把需要跳两遍的点再加一遍

在跳的过程中记录一下需要累加的次数即可。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=3e5+7;
int l[N];
int f[N][22],sum[N][22];
int n,m;
void init()
{
    f[n][0]=l[n],sum[n][0]=n-l[n];//画下图即理解,
    for(int i=n-1;i>=2;i--)//从n-1开始处理
    {
        f[i][0]=min(f[i+1][0],l[i]);//跳一步
        sum[i][0]=i-f[i][0];
    }
    for(int i=1;i<22;i++)
        for(int j=(1<<i);j<=n;j++)
        {
            f[j][i]=f[f[j][i-1]][i-1];//递推式
            sum[j][i]=sum[j][i-1]+sum[f[j][i-1]][i-1]+(1<<(i-1))*(f[j][i-1]-f[j][i]);
        }
    return;
}
int dist(int x,int y)//目标点，当前位置
{
    if(x>=l[y]) return y-x;//特判一步就能走到的
    int sp=1,ans=y-l[y];//先跳一步
    y=l[y];
    for(int i=20;i>=0;i--)
    {
        if(f[y][i]>=x)//还没跳到
        {
            ans+=sp*(y-f[y][i])+sum[y][i];//前缀和，有些数会被加sp遍，sum存的仅为走一步之和
            sp+=(1<<i);//走的这些都得加上
            y=f[y][i];//跳过来
        }
    }
    //下一次跳跳过了就停止转移，那么可能是刚好这一步跳到了，也可能还差一点
    if(y>x)//差一点，补上（类似倍增求LCA）
    {
        ans+=(sp+1)*(y-x);
    }
    return ans;
}
int gcd(int a,int b)
{
    return b>0?gcd(b,a%b):a;//辗转相除求gcd
}

int main()
{
    cin>>n;
    for(int i=2;i<=n;i++)
    {
        scanf("%d",&l[i]);
    }
    init();
    cin>>m;
    while(m--)
    {
        int L,R,X;
        scanf("%d%d%d",&L,&R,&X);
        int d=dist(L,X)-dist(R+1,X); //X为起点，从起点到左端点-起点到右端点+1（其实就是前缀和）
        int g=gcd(d,R-L+1);
        printf("%d/%d\n",d/g,(R-L+1)/g);
    }
    return 0;
}
